# Code of Conduct

## Our Standard

BeliefSync is an open-source project for builders, researchers, and practitioners working on coding agents and developer tooling. We want the project to feel serious, welcoming, and useful.

Participants are expected to:

- be respectful and constructive
- assume good intent while still giving clear technical feedback
- keep disagreements focused on ideas, design, evidence, and implementation details
- avoid harassment, personal attacks, and dismissive behavior

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, insults, or demeaning language
- deliberate intimidation
- discrimination or exclusionary conduct
- publishing private information without permission
- repeated bad-faith disruption of discussions

## Scope

This code of conduct applies in:

- GitHub issues
- pull requests
- project discussions
- official communication channels related to the repository

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, or other contributions that do not align with this code of conduct.
